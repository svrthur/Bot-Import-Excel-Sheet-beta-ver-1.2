# Telegram-бот для управления рекламным календарём

Бот позволяет отмечать рекламные кампании в Google Таблице прямо из Telegram: подсвечивает нужные ячейки, считает хронометраж по ТК и предупреждает о перегрузке.

**Таблица:** `https://docs.google.com/spreadsheets/d/17VeQQWTGotofrpNbUHDhUFhCc3qjLdwoesTxDDfJ7h4/edit`

---

## Что умеет бот

### 1. Подсветка кампании в таблице (`/start`)

Пошаговый диалог:

1. Отправьте `/start`
2. Введите номер строки — `118` или `A118`
3. Введите номера ТК через запятую — `203, 227, 230`

Бот отправит запрос в Google Apps Script, и ячейки в указанной строке под соответствующими ТК окрасятся в **зелёный** цвет (`#00ff00`). Сразу после этого автоматически пересчитывается строка 2 по текущей дате из ячейки **N7**.

---

### 2. Проверка порога 160 секунд

После каждой подсветки бот автоматически проверяет строку 2 (R2:GN2). Если в каком-либо ТК накоплено **больше 160 секунд**, бот пришлёт предупреждение:

```
⚠️ Внимание! В ТК № 203, 227 превышен порог в 160 секунд!
```

## Что умеет Apps Script

### Структура таблицы (конфигурация)

| Столбец | Содержимое |
|---|---|
| A | Название кампании |
| B | Тип |
| C | Длительность (секунды) |
| D | Владелец |
| E | Статус (`запланировано` / `опубликовано` / `завершено`) |
| F | Дата старта |
| G | Дата окончания |
| N7 | Выбранная дата для хронометража |
| R–GN (строка 1) | Заголовки ТК |
| R–GN (строка 2) | Итоговые секунды по каждому ТК |
| R–GN (строки 3+) | Ячейки кампаний: зелёные = активна на этом ТК, красные = истекла |

---

### Функции скрипта

#### `onEdit` — автопересчёт при изменении N7
Срабатывает автоматически каждый раз, когда вы меняете дату в ячейке **N7**. Пересчитывает строку 2 для всех ТК.

> Требует триггер: **On edit** → функция `onEdit`

---

#### `recalcTotalsRow_` — хронометраж по дате
Основная функция подсчёта. Для каждого ТК (столбцы R–GN) суммирует секунды из столбца **C** по всем строкам, где одновременно выполнены три условия:

- ячейка закрашена **зелёным** (`#00ff00`)
- дата из **N7** попадает в диапазон **[Дата старта (F) .. Дата окончания (G)]** включительно
- эффективный статус на дату N7 **не** «завершено»

Результат записывается в строку 2.

---

#### `recalcTotalsByN7` — ручной пересчёт
Запустите вручную в Apps Script для немедленного пересчёта строки 2 по текущему значению N7.

---

#### `recolorExpiredByToday` — перекраска истёкших кампаний
Проверяет все строки начиная с 3-й. Если сегодняшняя дата позже даты окончания (столбец G), зелёные ячейки этой строки перекрашиваются в **красный** (`#ff0000`).

> Требует триггер: **По времени → Ежедневно в 02:00**  
> Запустите `createDailyRecolorTrigger()` один раз — триггер создастся автоматически.

---

#### `effectiveStatusOnDate_` — эффективный статус
Определяет статус кампании на конкретную дату:
- если статус в столбце E один из трёх известных — используется он
- иначе вычисляется по датам: до старта → «запланировано», в диапазоне → «опубликовано», после окончания → «завершено»

---

#### `highlightCampaigns` — подсветка из бота
Принимает от бота данные вида `{ "234": ["203", "227"] }`, находит нужные столбцы по заголовкам строки 1 и красит ячейки зелёным. После этого автоматически вызывает пересчёт строки 2.

---

#### `hideFinishedCampaigns` — скрытие завершённых кампаний
Проверяет строки начиная со 120-й. Если статус в столбце E = «завершено» и с даты окончания (столбец G) прошло более 24 часов — строка скрывается.

> Требует триггер: **По времени → По часам** (например, каждый час)

---

#### `checkTKThreshold` — проверка лимита 160 с
Читает строку 2 (R2:GN2) и возвращает список ТК, где значение превышает 160 секунд. Используется ботом для отправки предупреждений.

---

#### `doPost` — приём команд от бота
Точка входа для POST-запросов. Принимает JSON-тело:
```json
{ "action": "highlight", "data": { "234": ["203", "227"] } }
```

---

#### `doGet` — API для бота и ручных запросов
| Параметр | Что делает |
|---|---|
| `?action=calculate` | Пересчитывает строку 2 по N7 и возвращает `{"status":"success"}` |
| *(без параметра)* | Возвращает список ТК с превышением 160 с: `{"status":"success","exceeded":["203"]}` |

---

#### `debugTKRow2` — диагностика нулей в строке 2
Запустите вручную, если строка 2 показывает нули. В логах (View → Logs) выведет:
- все имена листов и какой используется
- распознанная дата из N7
- все уникальные цвета, найденные в блоке R–GN
- строки, прошедшие фильтры по дате, статусу и цвету
- точную причину нулей, если данные не попадают в расчёт

## Настройка с нуля

### Шаг 1. Загрузите скрипт в таблицу

1. Откройте таблицу → **Расширения → Apps Script**
2. Полностью замените содержимое редактора кодом из файла `google-apps-script.js`
3. Нажмите **Сохранить** (Ctrl+S)

### Шаг 2. Разверните как Web App

1. **Deploy → New deployment**
2. Тип: **Web App**
3. Параметры:
   - Запуск от имени: `User deploying the web app` (я)
   - Кто имеет доступ: `Anyone` ← **обязательно**
4. Нажмите **Deploy**, скопируйте URL

### Шаг 3. Вставьте URL в бот

Откройте `server/bot.ts`, строка 7, вставьте скопированный URL:
```typescript
const APPS_SCRIPT_URL = 'https://script.google.com/macros/s/ВАШ_ID/exec';
```

### Шаг 4. Установите триггеры

В Apps Script → **Triggers (⏰)**:

| Функция | Тип триггера |
|---|---|
| `onEdit` | From spreadsheet → On edit |
| `recolorExpiredByToday` | Time-driven → Day timer → 02:00 (или запустить `createDailyRecolorTrigger()`) |
| `hideFinishedCampaigns` | Time-driven → Hour timer |

### Шаг 5. Убедитесь что в Replit установлен токен

В разделе Secrets (Replit) должна быть переменная `TELEGRAM_BOT_TOKEN` с токеном вашего бота от @BotFather.

## Решение проблем

| Симптом | Причина | Решение |
|---|---|---|
| Ячейки не красятся | Неверный URL в `bot.ts` или доступ не `Anyone` | Пересоздайте деплой, скопируйте новый URL |
| Строка 2 показывает нули | Функция не запускалась или данные не совпадают | Запустите `recalcTotalsByN7` вручную; при нулях — запустите `debugTKRow2` |
| Бот не отвечает (409 ошибка) | Запущены две копии бота | Перезапустите workflow в Replit (Stop → Run) |
| Красные ячейки вместо зелёных | Кампания истекла, сработала перекраска | Это штатное поведение — кампания завершена |
| `N7 не распознана как дата` | В N7 текст вместо даты | Установите формат ячейки N7 как «Дата» в Google Sheets |
